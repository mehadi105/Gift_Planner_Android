### PATTERN TO FOLLOW

Each screen follows this architecture:

```
Fragment (UI) 
  â†“ uses
ViewModel (business logic) 
  â†“ uses
Repository (data operations)
  â†“ uses

```

### STEP-BY-STEP FOR EACH SCREEN:

1. **Create the layout XML** (e.g., fragment_people.xml)
   - Add ScrollView/RecyclerView
   - Add form inputs
   - Add buttons

2. **Create item layout XML** (e.g., item_person.xml) for RecyclerView items

3. **Create RecyclerView Adapter** (e.g., PeopleAdapter.java)
   - Extend RecyclerView.Adapter
   - Implement ViewHolder
   - Handle item click callbacks

4. **Create ViewModel** (e.g., PeopleViewModel.java)
   - Extend ViewModel
   - Use MutableLiveData for UI state
   - Call repository methods
   - Handle callbacks and update LiveData

5. **Create ViewModelFactory** (e.g., PeopleViewModelFactory.java)
   - Implement ViewModelProvider.Factory
   - Inject repository dependencies

6. **Implement Fragment** (e.g., PeopleFragment.java)
   - Use ViewBinding
   - Get ViewModel with ViewModelProvider
   - Observe LiveData
   - Set up click listeners
   - Call ViewModel methods
